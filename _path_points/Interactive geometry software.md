<p><b>Interactive geometry software</b> (IGS, or dynamic geometry environments, DGEs) are <a href="/wiki/Computer_program" title="Computer program">computer programs</a> which allow one to create and then manipulate <a href="/wiki/Geometry" title="Geometry">geometric</a> constructions, primarily in <a href="/wiki/Euclidean_plane_geometry" title="Euclidean plane geometry" class="mw-redirect">plane geometry</a>. In most IGS, one starts construction by putting a few points and using them to define new objects such as <a href="/wiki/Line_(mathematics)" title="Line (mathematics)" class="mw-redirect">lines</a>, <a href="/wiki/Circle" title="Circle">circles</a> or other points. After some construction is done, one can move the points one started with and see how the construction changes.
</p>

<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=List_of_interactive_geometry_software&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The earliest IGS was the Geometric Supposer, which was developed in the early 1980s.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1"><span>[</span>1<span>]</span></a></sup> This was soon followed by Cabri in 1986 and The Geometer's Sketchpad.
</p>
<h2><span class="mw-headline" id="Comparison">Comparison</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=List_of_interactive_geometry_software&amp;action=edit&amp;section=2" title="Edit section: Comparison">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>There are three main types of computer environments for studying school geometry: supposers, dynamic geometry environments (DGEs) and <a href="/wiki/Logo_(programming_language)" title="Logo (programming language)">Logo</a>-based programs.<sup id="cite_ref-2" class="reference"><a href="#cite_note-2"><span>[</span>2<span>]</span></a></sup> Most are DGEs: software that allows the user to manipulate ("drag") the geometric object into different shapes or positions. The main example of a supposer is the Geometric Supposer, which does not have draggable objects, but allows students to study pre-defined shapes. Nearly all of the following programs are DGEs. For a related, comparative physical example of these algorithms, see <a href="/wiki/Lenart_Sphere" title="Lenart Sphere" class="mw-redirect">Lenart Sphere</a>.
</p>

<h3><span class="mw-headline" id="C.a.R.">C.a.R.</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=List_of_interactive_geometry_software&amp;action=edit&amp;section=15" title="Edit section: C.a.R.">edit</a><span class="mw-editsection-bracket">]</span></span></h3>

</p>
<h3><span class="mw-headline" id="CaRMetal">CaRMetal</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=List_of_interactive_geometry_software&amp;action=edit&amp;section=16" title="Edit section: CaRMetal">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a href="/wiki/CaRMetal" title="CaRMetal">CaRMetal</a> is a free <a href="/wiki/GNU_General_Public_License" title="GNU General Public License">GPL</a> software written in <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a>.
It is derived from C.a.R and provides another user interface.
</p>
<h3><span class="mw-headline" id="Cabri_II_Plus">Cabri II Plus</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=List_of_interactive_geometry_software&amp;action=edit&amp;section=17" title="Edit section: Cabri II Plus">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a rel="nofollow" class="external text" href="http://www.cabri.com">Cabri II Plus</a> is an extended analog of GSP and actually the ancestor of all DGSs.
</p>
<h3><span class="mw-headline" id="Cinderella">Cinderella</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=List_of_interactive_geometry_software&amp;action=edit&amp;section=18" title="Edit section: Cinderella">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a href="/wiki/Cinderella_(software)" title="Cinderella (software)">Cinderella</a> is very different from GSP, written in <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a>.
</p><p>The new version Cinderella.2 also includes a physics simulation engine and a scripting language. Also, it now supports macros, line segments, calculations, arbitrary functions, plots, etc. Full documentation is available online.
</p>
<h3><span class="mw-headline" id="Euklid_DynaGeo">Euklid DynaGeo</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=List_of_interactive_geometry_software&amp;action=edit&amp;section=19" title="Edit section: Euklid DynaGeo">edit</a><span class="mw-editsection-bracket">]</span></span></h3>

</p>
<h3><span class="mw-headline" id="Eukleides">Eukleides</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=List_of_interactive_geometry_software&amp;action=edit&amp;section=20" title="Edit section: Eukleides">edit</a><span class="mw-editsection-bracket">]</span></span></h3>

</p>
<h3><span class="mw-headline" id="Dr_Genius">Dr Genius</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=List_of_interactive_geometry_software&amp;action=edit&amp;section=21" title="Edit section: Dr Genius">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Dr Genius was an attempt to merge <a href="/wiki/Dr._Geo" title="Dr. Geo" class="mw-redirect">Dr. Geo</a> and the <a href="/wiki/Genius_(software)" title="Genius (software)" class="mw-redirect">Genius calculator</a>.

<h3><span class="mw-headline" id="Dr._Geo">Dr. Geo</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=List_of_interactive_geometry_software&amp;action=edit&amp;section=22" title="Edit section: Dr. Geo">edit</a><span class="mw-editsection-bracket">]</span></span></h3>

<h3><span class="mw-headline" id="GCLC">GCLC</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=List_of_interactive_geometry_software&amp;action=edit&amp;section=24" title="Edit section: GCLC">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><b>GCLC</b><sup id="cite_ref-4" class="reference"><a href="#cite_note-4"><span>[</span>4<span>]</span></a></sup> is a dynamic geometry tool for visualizing and teaching geometry, and for producing mathematical illustrations. In GCLC figures are described rather than drawn. This approach stresses the fact that geometrical constructions are abstract, formal procedures and not figures. A concrete figure can be generated on the basis of abstract description. There are several output formats, including LaTeX, LaTeX/PStricks, LaTeX/Tikz, SVG, PostScript. There is a built-in geometry theorem prover (based on the area method). GCLC is available for Windows and Linux. WinGCLC is a Windows version of GCLC with graphical interface and provides a range of additional functionalities.
</p>

<li> <b>Algebra</b>: direct input of inequalities, implicit polynomials, linear and quadratic equations; calculations with numbers, points and vectors</li>
<li> <b>Calculus</b>: direct input of functions (including piecewise-defined); intersections and roots of functions; symbolic derivatives and integrals (built in CAS); sliders as parameters</li>

<h3><span class="mw-headline" id="Geometria">Geometria</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=List_of_interactive_geometry_software&amp;action=edit&amp;section=56" title="Edit section: Geometria">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a rel="nofollow" class="external text" href="http://geocentral.net/geometria">Geometria</a> is free (<a href="/wiki/MIT_License" title="MIT License">MIT License</a>) cross-platform software based on a two-role (teacher, student) model. The teacher creates a problem, provides it with an answer and saves the problem in a file. The student opens the file and solves the problem. The student's actions are recorded. The teacher plays back the solution and grades it. The figures are limited to convex polyhedra, which can be measured, drawn upon, transformed, cut and joined.
</p>

<h2><span class="mw-headline" id="Continuity_vs._determinism">Continuity vs. determinism</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=List_of_interactive_geometry_software&amp;action=edit&amp;section=66" title="Edit section: Continuity vs. determinism">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>All these programs can be divided into two category: deterministic and continuous.
GeoGebra can be deterministic or continuous (one can change it in preferences).
</p><p>All constructions in the deterministic programs (GSP, Cabri, Kseg and most of others) are completely determined by the given points but the result of some constructions can jump or behave unexpectedly when a given point is moved.
</p><p>On the contrary, some constructions in continuous programs (so far only Cinderella and GeoGebra), depend on the number of hidden parameters and in such a way that moving a given point produces a continuous motion of the construction, as a result, if the point is moved back to the original position the result of construction might be different.
</p><p>Here is a test to check whether a particular program is continuous:
</p><p>Construct the <a href="/wiki/Orthocenter" title="Orthocenter" class="mw-redirect">orthocenter</a> of triangle and three midpoints (say <i>A', B' C' </i> ) between vertices and orthocenter.
</p><p>Construct a <a href="/wiki/Circumcircle" title="Circumcircle" class="mw-redirect">circumcircle</a> of <i>A'B'C' </i>.
</p><p>This is the <a href="/wiki/Nine-point_circle" title="Nine-point circle">nine-point circle</a>, it intersects each side of the original triangle at two points: the base of altitude and midpoint. Construct an intersection of one side with the circle at midpoint now move opposite vertex of the original triangle, if the constructed point does not move when base of altitude moves through it that probably means that your program is continuous.
</p><p>Although it is possible to make a deterministic program which behaves continuously in this and similar simple examples, in general it can be proved that no program can be continuous and deterministic at the same time.<sup id="cite_ref-16" class="reference"><a href="#cite_note-16"><span>[</span>16<span>]</span></a></sup>
</p>
